# 1. Record architecture decisions

- Status: accepted
- Date: 2026-06-08

## Context
We need a lightweight, durable record of *why* key choices were made, for future
contributors (human and AI).

## Decision
Use Architecture Decision Records (ADRs), one Markdown file per decision in
`docs/adr/`, numbered sequentially. Keep them short. Supersede rather than delete.

## Consequences
A traceable decision history that newcomers can read in minutes.
